## 時系列予測の新手法で、短期から長期まで全部勝てるようになりました

時系列予測って、短期に強いモデルを作ると長期で精度が崩壊するし、長期向けに作ると短期が弱くなる、というジレンマがずっとありました。

今回作った手法（FLAR v9）は、このジレンマを解消できました。発想はシンプルで、「1時間ごとに168回予測する」のをやめて、「1日の合計を7回だけ予測して、時間帯ごとの配分パターンを掛ける」というものです。

たとえば電力需要なら、「明日は合計いくら使うか」だけ当てて、「朝は全体の15%、昼は25%、夜は30%...」という配分はいつもほぼ同じなのでそのまま使います。予測するものが1本の滑らかな系列になるので、予測が簡単で安定します。

98データセットで検証した結果（1.0が基準手法と同等）：

| 予測期間 | 従来最良 | v9 |
|---|---|---|
| 短期 | 0.884 | **0.881** |
| 中期 | 1.053 | **0.877** |
| 長期 | 1.418 | **0.925** |

従来手法は中長期で基準を超えて悪化していたのが、v9は全期間で基準を下回りました。深層学習は使っていません。

技術的に気に入っているのは、予測モデルが線形回帰1個だけで済むところです。SVD 1回で係数が求まって、ハイパーパラメータも自動決定されるので、学習ループもGrid Searchもいりません。168回の再帰予測が7回に減るので誤差が溜まりにくく、それが長期でも崩壊しない理由です。実装もNumPyだけの約150行で、この軽さで98データセット横断の性能が出るのはなかなか面白いと思います。

## Slack用（技術者の後輩向け）

FLAIR (Forecasting via Level-Aggregate Inversion with Ridge) っていう時系列予測手法でめちゃくちゃ綺麗なのができたので聞いてほしい。

まず問題設定。時系列 y(1), y(2), ..., y(T) から H ステップ先まで予測したい。普通は y(t+1) = f(y(t)) を再帰的に回すんだけど、H=168（1週間×1時間刻み）とかになると誤差が複利で膨らんで精度が崩壊する。これが時系列予測の長期問題。

FLAIRはこれを5ステップで解く。

**Step 1: Reshape — 時系列を行列に折り畳む**

時系列を周期 P で折り畳んで行列にする。`Y = reshape(y, (n, P))` 。1時間刻みなら P=24 で、各行が「1日分」、各列が「ある時間帯の全日分」になる。

**Step 2: Decompose — Level × Shape に分解する**

この行列を rank-1 近似する。

```
Y(i, j) ≈ L(i) × S(j)
```

L(i) は i 日目の合計（Level）、S(j) は j 時の配分比率（Shape）。やってることは行列の列和と、列を列和で割る正規化。これだけ。Shapeは直近数期間の平均で固定する。すると予測すべきは L(i) という1本のスカラー時系列だけになる。元は P 本の系列を同時に予測する問題だったのが、1本に帰着する。

**Step 3: Transform — Level を Box-Cox + NLinear で正規化する**

Lに対してBox-Cox変換で分布を安定化し、最終観測値を引く（NLinear正規化）。元系列に直接Box-Coxをかけると各phaseでスケールが不整合になる問題があったけど、Lは常に正の1本の系列だから整合的。

**Step 4: Predict — Ridge 回帰を SVD で 1 回解く**

正規化した L に対して Ridge 回帰を解く。ここがちょっと面白くて、L は「日単位」の系列になっているので、特徴量も日単位の世界で設計する。元の時系列で「日周期」を捉えるために必要だった cos(2πt/24) とかの主周期Fourier項は、reshape した時点で構造的に解決済みなので不要。Ridge が学ぶのは「週周期」や「年周期」など、P より長い二次周期だけ。たとえば元が1時間刻み (P=24) なら、Level 空間での週次周期は 168/24=7 になるので、特徴量は `[1, t/n, cos(2πt/7), sin(2πt/7), lag-1]` みたいにコンパクトになる。一番重い主周期をFourier近似じゃなくreshapeで「構造的に消す」のが設計のキモ。

SVD 経由の閉形式解で、正則化パラメータ α は GCV（Leave-One-Out 交差検証の閉形式近似）で自動決定。学習ループなし、ハイパラチューニングなし。行列分解1回で全部出る。

予測は L を m = ceil(H/P) ステップだけ再帰。H=168 の再帰が m=7 に圧縮される。

**Step 5: Reconstruct — L̂ × S で元の粒度に復元する**

予測した L̂ に Box-Cox 逆変換をかけて、Step 2 の Shape S を掛けて元の時間粒度に復元する。

```
ŷ(k, j) = BC⁻¹(L̂(k)) × S(j)
```

---

**なぜこれが効くか**

L は元系列の P 点合計だから分散が 1/P。再帰回数も H/P。つまり「再帰回数が 1/P」×「1回あたりの誤差分散が 1/P」で、長期予測の誤差蓄積が O(1/P²) のオーダーで減る。これが長期でも崩壊しない構造的な理由。

結果、98データセット横断で全予測期間でSeasonalNaiveを下回った。

```
           従来最良   FLAIR
短期       0.884     0.881
中期       1.053     0.877   ← ここが一番やばい
長期       1.418     0.925
```

中期は従来手法が基準を超えて悪化してた領域で、FLAIRはむしろ短期より良い。意味がわからない。

全部NumPy約150行。GPUなし。深層学習なし。reshape, sum, svd, solve。線形代数だけでここまで出るのはだいぶ気持ちいい。

## みんな向け説明（簡潔版）

FLAIR（Forecasting via Level-Aggregate Inversion with Ridge）は、「周期構造による時系列のrank-1 tensor分解で、予測問題を集約レベルの単一系列回帰に帰着させる手法」です。

たとえば、k個のセンサーが1時間刻みで記録している時系列データがあり、1週間先（168ステップ）まで予測したいとします。素朴にやると k × 168 回の予測が必要で、1ステップごとに誤差が蓄積するので、長期になるほど精度が崩壊します。

FLAIRはこの問題を、時系列の構造を利用して劇的に圧縮します。

まず、各系列を24時間（=日周期）で折り畳んで行列にします。すると各系列が「1日の総量 L(i)」と「時間帯ごとの配分パターン S(j)」に分解できます。さらに複数のセンサーについても「センサーごとのシェア S_var(v)」で分解すると、最終的に予測すべきは「全センサー × 全時間帯の合計」という、たった1本のスカラー時系列だけになります。

```
Y(v, i, j) ≈ L(i) × S_var(v) × S_phase(j)

もとの予測問題:  k × 168 回の再帰予測（誤差が複利で膨張）
FLAIR:          1本 × 7回の再帰予測（最も滑らかな系列に対して）
```

この集約された1本の系列は全変量・全時間帯のノイズが平均化されて極めて滑らかなので、線形回帰1つで高精度に予測できます。予測後は配分パターンを掛け戻すだけで元の粒度に復元されます。

結果として、98データセット横断で、短期〜長期の全予測期間で基準手法を上回りました。特に、従来手法が崩壊していた中長期予測で大きな改善が出ています。深層学習は使わず、NumPy約150行で実装できます。

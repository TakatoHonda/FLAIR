# FLAIR: 周期的時系列はランク1行列である

**Factored Level And Interleaved Ridge**

Mellon Inc. テクニカルレポート — 2026-03-23

## 1. 問い

まず、この問いから始めたい:

> **なぜ数十億パラメータの Foundation Model が、SVD 1回・ハイパーパラメータ 0個の手法に負けるのか？**

GIFT-Eval ベンチマーク（97設定、23データセット、7ドメイン）において、FLAIR は relMASE=0.920、relCRPS=0.690 を達成し、全統計的手法を上回り、全ホライズンで Seasonal Naive に勝利する（short=0.900, medium=0.929, long=0.965）。ノートPC上で29分。GPU不要。学習ループ不要。モデル選択不要。

その答えは、周期的時系列に関する構造的な観察にある。一度見たら、もう見えなくなることはない。

![Benchmark](fig4_benchmark.png)

## 2. 観察: リシェイプが構造を明らかにする

任意の周期的時系列 — 時間単位の電力消費、日単位の売上、月単位の入院数 — を**主周期でリシェイプ**する。時間単位の系列で周期 P=24 なら、24行（時間帯）× n_days列 の行列になる。

その結果を見てみよう:

![Rank-1 Structure](fig1_rank1_structure.png)

3つのことが見て取れる:

1. **行列に構造がある** — ピーク時間帯は明るい帯、夜間は暗い帯。これが日内の「Shape（形状）」。
2. **列同士が似ている** — 各日は概ね同じパターンで、スケールだけが異なる。これが日間の「Level（水準）」。
3. **SVDが裏付ける** — 第1特異値が支配的。σ₁/σ₂ ≈ 20。行列は近似的に**ランク1**。

これは偶然ではない。周期的時系列の本質的な性質である:

> **周期的時系列は、その周期でリシェイプすると、近似的にランク1行列になる。**

数学的に:

```
mat(phase, period) ≈ σ₁ · u₁ · v₁ᵀ = Shape(phase) × Level(period)
```

Shape は周期内パターン（どの時間帯が高い/低い）を、Level は周期間ダイナミクス（どの日が高い/低い）を捉える。

---

## 3. アイデア: 振幅を予測し、形状を凍結する

行列がランク1であるなら、P × n_complete 個すべてを予測する必要はない。**1つの系列** — Level — を予測し、固定された Shape を掛ければよい。

![Level × Shape](fig2_level_shape.png)

アイデアの全体はこれだけ:

```
y_hat(phase, period) = Level_hat(period) × Shape(phase)
```

- **Shape** = 直近 K=5 周期の周期内比率の平均。学習しない。最適化しない。データから単純に平均する。
- **Level** = 周期合計。Box-Cox変換、NLinear正規化（最終値を引く）を経て、ソフト平均GCV付きRidge回帰で予測。

1つの式。1回のSVD。ハイパーパラメータ0個。

---

## 4. なぜこれが機能するか: 3つの圧縮

FLAIR は3つの圧縮を同時に達成し、それぞれが独立に予測精度を改善する:

### 圧縮 1: n → n/P サンプル（ノイズ低減）

Level系列は、各周期内の全 P フェーズの**合計**である。中心極限定理により、P 個の値を合計するとノイズは √P 減少する。時間データ（P=24）の場合、Level系列は生系列より**5倍ノイズが少ない**。

これは理論的議論にとどまらない。FLAIR が生系列を直接扱う手法に勝つ理由を説明する — それらは同じデータ量で5倍のノイズと戦っている。

### 圧縮 2: H → m = ⌈H/P⌉ 予測ステップ（安定性）

初期手法（V5）は2週間の時間単位予測で H=336 ステップを再帰的に予測した。各ステップが予測誤差をフィードバックし、誤差が指数的に蓄積する。

FLAIR は Level系列上で m = ⌈336/24⌉ = **14ステップ**を予測し、Shape を掛けて336値を復元する。336ステップの代わりに14ステップ — 誤差蓄積は24倍軽微。

### 圧縮 3: Shape は構造的（過学習への免疫）

Shape ベクトルはデータ比率から推定され、Ridge モデルで学習されない。これは:
- Ridge が推定する**パラメータが少ない**（Level特徴量のみ、約6列）
- Shape は過学習できない — 経験的平均であり、最適化の結果ではない
- Ridge のペナルティ α → ∞ のとき、予測 → Seasonal Naive（最終周期の繰り返し）

最後の点は微妙だが重要: 最大正則化時の FLAIR は**最強の単純ベースラインである Seasonal Naive に退化する**。通常の Ridge は特徴量付きではゼロに退化し、無意味である。

## 5. アルゴリズム

![Pipeline](fig5_pipeline.png)

```
入力:  y(t), 予測ホライズン H, 周波数 freq

1. カレンダー参照: P, 副周期を FREQ_TO_PERIODS テーブルから取得
   例: H → [24, 168]（日次 + 週次）

2. リシェイプ: mat = y.reshape(n_complete, P).T    → (P, n_complete)

3. Shape:   S = mean(proportions[:, -5:])         → 構造的、学習しない

4. Level:   L = mat.sum(axis=0)                   → 1つの滑らかな系列
            L_bc = BoxCox(L + 1, λ_MLE)           → 分散安定化
            L_innov = L_bc - L_bc[-1]              → NLinear正規化

5. 特徴量: X = [1, trend, Fourier(secondary/P), lag₁, lag_cp]

6. Ridge SA: β = 25個のRidge解のsoftmax加重平均
             （1回のSVD、GCV最適α平均化）

7. 予測:   L_hat を m = ⌈H/P⌉ ステップ予測（再帰的）
            y_hat = L_hat × S                     → 復元

8. SVD残差量子化: E = M − L×Sᵀ → 位相別相対残差R
   シナリオ整合サンプリング（同一stepの全位相が同一歴史列を共有）
   sample = L_path × S × (1 + phase_noise) → 予測区間
```

推定パラメータ数: 約6（切片 + トレンド + Fourier 2つ + ラグ 1-2）。
チューニングするハイパーパラメータ: **0個**（αはGCV-SAでデータ駆動、K=5は固定）。

---

## 6. ソフト平均GCV: αフリーなRidge

標準的な Ridge は GCV を最小化する1つの α を選択する:

```
α* = argmin GCV(α)
```

これは離散的な選択であり、2つの α が同程度の GCV を持つ場合、選択が不安定になる。

FLAIR は**ソフト平均GCV**を使用する: 最良の α を選択する代わりに、**全25個のRidge解**のsoftmax加重平均を取る:

```
w_i = softmax(−GCV_i / GCV_min)
β_SA = Σ w_i · β_i
```

1つの α が支配的なら、w ≈ δ（argmin と同等）。GCV曲線が平坦なら、複数の α が寄与する — 自動的な正則化平均。

すべて1回のSVDから: U, s, Vt = SVD(X)。各 α は O(p) 演算のため、25候補で 25 × O(p) — 無視できるコスト。

## 7. 凍結Shape定理（経験的）

FLAIR 開発後、改善のために多大な労力を費やした。Shape を適応的、学習可能、またはコンテキスト依存にする**14の異なるアプローチ**を4カテゴリにわたってテストした:

![Frozen Shape](fig3_frozen_shape.png)

**すべての試みが精度を悪化させた。**

| カテゴリ | 試行内容 | 最良結果 |
|---------|---------|---------|
| Shape適応 | EWMA重み付け、Fourierフィッティング、ランク-r SVD（r=2,3）、指数平滑 | すべて > 1.05 |
| 正則化変更 | James-Stein縮約、曲率正則化Ridge、ラグ減衰 | すべて > 1.10 |
| アーキテクチャ変更 | 季節性除去先行パイプライン、多変量テンソルプーリング、階層的リコンシリエーション | すべて > 1.10 |
| 不確実性 | フェーズ別区間、ホライズンスケールバンド | > 1.15 |

結論は経験的に頑健であり、直感に反する:

> **Shape は学習すべきでない。K=5の生比率の平均が最適。**

なぜか？ Shape 推定は**観測数が非常に少ない状況でのバイアス-バリアンスのトレードオフ**（K=5周期）だからである。単純平均より複雑な手法 — Fourierフィッティング、指数重み付け、SVD、Ridge埋め込みShape — はバイアスを除去するより多くのバリアンスを導入する。単純平均はこのサンプルサイズにおける**最小分散不偏推定量**である。

これは FLAIR の最も深い経験的発見であり、実用的な含意がある: **Shape を改善しようとしてはいけない**。Level Ridge、Conformal層、周期検出を改善すべきだが、Shape はそのままにすべきである。

## 8. マルチ周期カレンダーテーブル

FLAIR はデータ駆動の周期検出ではなく、ハードコードされたカレンダーテーブルを使用する:

```python
FREQ_TO_PERIODS = {
    'H':   [24, 168],     # 日次 + 週次
    '5T':  [12, 288],     # 時間次 + 日次
    '15T': [4, 96],       # 時間次 + 日次
    'D':   [7, 365],      # 週次 + 年次
    'W':   [52],           # 年次
    'M':   [12],           # 年次
    ...
}
```

主周期 P（最初のエントリ）がリシェイプを決定する。副周期は Level Ridge のためのクロス周期 Fourier 特徴量を提供する。

なぜデータから周期を検出しない（ACF, FFT）のか？ それは:
1. 短い系列での ACF/FFT は偽の周期を検出する（テスト済み — FFTスペクトル周期は悪化した）
2. カレンダー周期は**常に正しい**（1日 = 24時間、1週 = 7日）
3. パラメータなし、閾値なし、計算なし

これはルックアップテーブルとして表現されたドメイン知識。最も単純な周期検出であり、最も良く機能する。

## 9. FLAIR でないもの

FLAIR の機能を理解するには、何を避けているかを理解することが有用:

| 他の手法 | FLAIR | 理由 |
|---------|-------|------|
| 季節パターンを学習 | データからShapeを固定 | K=5では学習に不十分 |
| 複数モデル + 選択 | 1つのRidge、1回のSVD | 選択はバリアンスを増やす |
| 主周期にFourier特徴量 | 主周期でリシェイプ | 構造 > 計算 |
| 生系列で再帰予測 | Level上で再帰（mステップ） | 誤差蓄積ステップが24分の1 |
| ニューラルネットワーク | 閉形式のRidge | 勾配なし、エポックなし、GPU不要 |
| ハイパーパラメータチューニング | GCVソフト平均（データ駆動） | チューニングパラメータ0個 |

## 11. 理論的位置づけ

FLAIR は**トップダウン時間的分解**（Athanasopoulos & Kourentzes, 2017）として理解できる:

- 「上位レベル」は Level系列（周期合計）
- 「下位レベル」は生時系列
- 分解は固定比率（Shape）で行う

また、**乗法的季節分解**（X-11スタイル）としても理解できる:

- 季節指数 = Shape
- 季節調整済み系列 = Level
- トレンド予測 = Level上のRidge

新規性は個々のコンポーネントにはなく、以下を同時に達成する特定の組み合わせにある:
1. ハイパーパラメータ0個
2. SVD 1回
3. Foundation Model と競合する精度
4. Shape は学習すべきでないという経験的証明

## 13. FLAIR が失敗するとき

透明性のために弱点を認める:

1. **非周期的系列**（m4_yearly, A周波数）: V5（Fourier-ラグRidge）にフォールバックし、これは弱い
2. **非常にノイジーな多変量**（bitbrains_fast_storage/H）: Shape平均が変量固有のパターンをぼかす
3. **系統的に変化するShape**（平日 vs 週末）: 固定Shapeは平均を捉え、モードを捉えない

これらは実装上のバグではなく、真の限界である。将来の研究方向（時変Shape、クロス変量学習）を指し示すが、我々の徹底的なアブレーション研究はこれらが**まだ解決されていない**ことを示している。

# 周期性自動検出 × 階層予測 reconciliation の調査

調査日: 2026-03-17

## 概要

周期性自動検出と階層時系列予測（reconciliation）の交差領域を調査した。**結論: この交差領域はほぼ空白であり、高い新規性がある。**

## 2. 周期性ベースのクラスタリング: 驚くほど未開拓

### 現状

**時系列を周期特性でクラスタリングした研究はほぼ存在しない。**

- DUET (KDD 2025, arXiv: 2412.10859): 周波数ドメインでチャネル関係を捉えるが、周期特性によるグループ化ではない
- Zhang et al. (2024): 生系列、DTW、統計特徴量でクラスタリングしたが、**周期性特徴は使用していない**
- 一般的なTS clustering: shape-based (DTW), feature-based (統計量), model-based (ARIMA距離) → **frequency-based は未活用**

### ギャップ

> 「周期=7の系列」「周期=7+365の系列」「周期=24+168の系列」のように、検出された周期特性でクラスタリングし、それを階層構築に使う研究は**存在しない**。

## 3. 階層予測における周期性/季節性の扱い

### 集約と季節性の関係

- 日次系列を週次に集約 → 日周期（period=7）は消滅
- 時間次系列を日次に集約 → 日内パターンは消滅
- **この現象は概念的に理解されているが、reconciliationで明示的にモデル化されていない**

### reconciliationは季節性を無視

- MinT, WLS, OLS, MinTrace: すべて**季節性に無関心**
- 共分散行列Wの推定に季節構造を反映していない

### 唯一の例外: HiFoReAd (Yang et al., 2024)

- arXiv: 2412.14718
- "Multi-Stage Hierarchical Forecasting Reconciliation and Adjustment"
- **harmonic alignment** と **stratified scaling** で季節性を保存
- 3-40%改善を報告
- ただし**自動周期検出は使用していない**

## 4. Foundation Modelと周期性

### FMは周期性を暗黙的にしか扱えない

すべての主要FM（Chronos, TimesFM, Moirai）は事前学習データからパターンを暗黙的に学習。明示的な周期検出・季節分解は行わない。

### FMの周期性における弱点

**"Evaluating TSFMs on Noisy Periodic Time Series" (2025, arXiv: 2501.00889):**

FMが劣化する条件:
- **長い周期**
- **高ノイズ**
- **低サンプリングレート**
- **複雑な波形**

→ 周期性を明示的に扱う手法が有利になる領域。

**追加エビデンス:**
- arXiv: 2509.19465: 統計モデルがFMを sCRPSで>8%, MASEで>20% 上回る
- FMはコンテキストウィンドウが短い場合に部分的な季節パターンを利用できるが、これは転移学習の恩恵であって周期性モデリングの優位性ではない

## 5. 新規性が高い研究方向

### ギャップ1: 周期性で階層構築（★★★★★）

```
各系列の周期性を自動検出（FFT/ACF/TimesNetなど）
  → 周期特性で階層クラスタリング
    例: [period=7のみ] → [period=7+30] → [period=7+30+365]
  → 各クラスタレベルで予測
  → MinT reconciliation
```

**なぜ効くか:**
- 同じ周期特性を持つ系列は予測誤差の相関構造が似る → MinTの共分散推定が改善
- Zhang et al.は「構造が重要」と示した → 周期性に基づく構造は「意味のある構造」
- FMが苦手な長周期・複雑周期で統計手法が有利

**Zhang et al.のランダム実験への反論:**
- 彼らは汎用的なクラスタリング（DTW, Euclidean）を使用 → 「似た系列をまとめても効かない」
- 周期性クラスタリングは「似た」ではなく「予測モデルの共有可能性」でグループ化 → 異なる仮説

### ギャップ2: 周期性を考慮したreconciliation（★★★★☆）

- 周期成分ごとにreconciliationを分離
- 周波数ドメインでreconciliation
- 集約レベルごとに存在する季節成分を明示的にモデル化

### ギャップ3: 周期性aware cross-temporal reconciliation（★★★☆☆）

- 現在のcross-temporal手法は固定的な集約スケジュール（時間→日→週→月）
- 検出された周期に基づいて集約レベルを選択する手法は不存在

### ギャップ4: FM + reconciliation + 周期性の統合（★★★★★）

- FMは長周期・複雑周期で劣化
- reconciliationでこの弱点を補償できる可能性
- この3要素を組み合わせた研究は**ゼロ**

### ギャップ5: 多季節性ベンチマーク（★★★☆☆）

- 複数の共存する季節性を正しく識別・予測できるかを評価するベンチマーク自体が不存在
- SarSim0 (arXiv: 2601.00970) は合成データ生成ツールだが評価ベンチマークではない

---

## 6. 提案するストーリーライン

> **「Foundation Modelは周期性を暗黙的にしか扱えず、複雑な周期パターンで劣化する。我々は周期性の自動検出を用いて意味のある階層構造を構築し、MinT reconciliationと組み合わせることで、FMが苦手とする領域で系統的に上回る。」**

### 新規性の3本柱

1. **周期性ベースの階層構築** — 誰もやっていない
2. **FMとの体系的比較** — 階層予測 vs FM の比較研究がない
3. **FMの既知の弱点を突く** — 長周期・高ノイズ・複雑波形

### リスクと検証すべき仮説

- 周期性クラスタリングがランダムより良い構造を作ることを実験で示す必要あり
- 周期性検出自体の精度がパイプライン全体に影響
- 小規模実験で「周期性クラスタリング vs ランダム vs Ward法」を比較すべき

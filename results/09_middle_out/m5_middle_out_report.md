**実験日**: 2026-03-17
**データセット**: M5 Accuracy Competition (m5_standard.csv) — 1,000系列ランダムサンプル
**実行環境**: macOS Darwin / Apple M4 / CPU逐次処理 / GPU不使用 / RAM 24GB / seed=42

## 1. 実験概要

### 1.1 動機

v1s（ネスト+構造的WLS）がベスト手法と確定したが、パイプライン全体の92%（317s/536s）を
P1のper-series MSTL+ETS予測が占めていた。本レポートでは、P1を完全にスキップし、
**L2クラスタレベル予測を歴史的シェアで各SKUに按分する「Middle-out」アプローチ**で
高速化と精度維持の両立を検証する。

### 1.2 Middle-outの原理

```text
従来 (v1s):
  P1: 1,000系列個別予測 (317s) → 階層予測 (27s) → reconciliation (0.01s) = 344s

Middle-out:
  階層予測 (27s) → L2予測を歴史的シェアで按分 → reconciliation (0.01s) = 27s
```

Bottom-level予測をper-series MSTLで生成する代わりに、L2クラスタの集約予測を
訓練期間の平均需要比率で各SKUに分配する。これにより、1,000回の個別MSTL+ETS計算が
不要になる。

### 1.3 歴史的シェアの計算方法

```python
for each L2 cluster c:
    cluster_means = mean_demand[series in c]     # 訓練期間の系列別平均需要
    shares = cluster_means / sum(cluster_means)   # 需要比率
    bottom_pred[series in c] = shares × L2_forecast[c]  # 按分
```

ゼロ需要クラスタ（cluster_total < 1e-12）は均等按分にフォールバック。

### 3.4 MOの精度特性の解釈

MOが中央値では改善するが個別改善率が低い（58.9% vs 88.0%）理由:

1. **按分の性質**: 歴史的シェアは固定比率のため、需要パターンが変化した系列には追随できない
2. **外れ値への感度**: 大きく外す系列がある一方、多数の系列では穏やかに改善
3. **WAPE平均 vs 中央値の乖離**: 平均(1.79)は中央値(1.30)より大きく、一部系列が大きく悪化していることを示す

1. **L2クラスタ予測の安定性**: 個別系列のノイズが集約により平滑化され、予測精度が向上

## 5. 限界と注意点

### 5.1 個別系列の精度

- **改善率が低い（58.9%）**: 全系列の約4割で精度が悪化
- 需要パターンが訓練期間から大きく変化した系列では、歴史的シェアが不正確になる
- **対策候補**: 直近N日の移動平均シェア、指数減衰シェアの検討

### 5.2 WAPE平均の悪化

- WAPE平均は1.7937でv1s（1.5959）より+12.4%悪化
- 少数の系列が大きく外すことが原因（中央値は改善、平均は悪化）
- **対策候補**: 外れ値系列の検出と個別予測へのフォールバック

### 5.3 特徴量抽出の依存性

- 現行スクリプトではP1がクラスタリング用特徴量（22次元）の抽出にも使われている
- P1を完全スキップするには、特徴量抽出を別手法に置き換える必要がある
- **対策候補**: 集約統計量（mean, std, zero_ratio, ACF等）のみでクラスタリング

### 5.4 シェアの静的性質

- 歴史的シェアは訓練期間全体の平均であり、トレンド変化に対応できない
- 季節性の異なる系列が同一クラスタに入ると、シェアが不正確になる

### C. Middle-outのアルゴリズム詳細

```
入力:
  - train_matrix: (N, T_train) 訓練行列
  - fc_L2_v1: {cluster_id: (H,)} L2クラスタ予測
  - labels_L2_v1: (N,) L2クラスタラベル
  - S_v1, W_struct_v1: summingマトリクスと構造的WLS重み

Step 1: 歴史的シェア計算
  series_mean_demand = train_matrix.mean(axis=1)  # (N,)
  for each cluster c:
    shares[c] = mean_demand[series in c] / total_mean_demand[c]

Step 2: L2予測を按分
  for each cluster c:
    bottom_pred[series in c] = shares[c] × fc_L2[c]

Step 3: y_hat構築
  y_hat[bottom] = bottom_pred (按分結果)
  y_hat[L2] = fc_L2 (クラスタ予測)
  y_hat[L1] = fc_L1 (ティア予測)
  y_hat[L0] = fc_L0 (合計予測)

Step 4: MinT reconciliation
  b_tilde = (S'W⁻¹S)⁻¹ S'W⁻¹ y_hat

Step 5: Clamp
  pred = max(0, b_tilde)
```
